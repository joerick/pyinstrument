# 输出与渲染模块需求说明文档

## 模块范围

本模块对应 `pyinstrument` 项目中的输出与渲染相关功能，主要覆盖以下源码文件：

- `pyinstrument/renderers/base.py`
- `pyinstrument/renderers/console.py`
- `pyinstrument/renderers/html.py`
- `pyinstrument/renderers/jsonrenderer.py`
- `pyinstrument/renderers/speedscope.py`
- `pyinstrument/renderers/pstatsrenderer.py`
- `pyinstrument/renderers/session.py`
- `pyinstrument/renderers/__init__.py`
- `pyinstrument/processors.py`
- `pyinstrument/renderers/html_resources/app.js`
- `pyinstrument/renderers/html_resources/app.css`

这些文件共同承担“将采样得到的性能会话数据加工并输出为不同展示形式”的职责，是用户直接感知 `pyinstrument` 结果的核心模块。

## 模块目标

输出与渲染模块的总体目标是：

1. 接收 `Profiler` 生成的 `Session` 数据。
2. 根据不同使用场景，将性能调用树渲染成适合终端、浏览器、JSON 工具链、Speedscope、pstats 等形式的输出。
3. 在输出前对调用树进行裁剪、聚合、隐藏、分组、过滤等预处理，提升可读性与可用性。
4. 保证输出结果在空会话、大会话、跨平台路径、不同显示参数下行为稳定。

## 主要功能点

### 渲染器抽象基类

对应文件：`pyinstrument/renderers/base.py`

功能点：

- 定义 `Renderer` 抽象基类，约束所有渲染器都提供 `render(session)` 方法。
- 定义 `FrameRenderer`，为基于调用树渲染的渲染器提供统一预处理框架。
- 支持 `show_all` 与 `timeline` 两类通用渲染选项。
- 统一维护 `processors` 与 `processor_options`。

代码引用：

```python
class Renderer:
    def render(self, session: Session) -> str:
        raise NotImplementedError()
```

```python
def preprocess(self, root_frame: Frame | None) -> Frame | None:
    frame = root_frame
    for processor in self.processors:
        frame = processor(frame, options=self.processor_options)
    return frame
```

### 控制台文本渲染

对应文件：`pyinstrument/renderers/console.py`

功能点：

- 生成终端文本输出。
- 支持树状输出和 `flat` 平铺输出。
- 支持 `unicode`、`color`、`short_mode` 等显示选项。
- 支持按秒数或总时间百分比展示时间。
- 支持对库代码进行隐藏分组显示。
- 支持按总耗时比例过滤掉极小节点。

用户价值：

- 适合命令行快速查看。
- 输出直观，便于在终端、日志、CI 中阅读。

代码引用：

```python
if self.flat:
    result += self.render_frame_flat(self.root_frame, precision=precision)
else:
    result += self.render_frame(
        self.root_frame, precision=precision, indent=indent, child_indent=indent
    )
```

### HTML 渲染

对应文件：`pyinstrument/renderers/html.py`、`html_resources/app.js`、`html_resources/app.css`

功能点：

- 输出交互式 HTML 页面。
- 将会话数据与前端资源打包进单文件 HTML。
- 对超大采样数的会话执行自动重采样，避免浏览器卡顿。
- 支持将 HTML 输出到临时文件并自动在浏览器中打开。
- 提供 `JSONForHTMLRenderer`，为前端页面准备专用 JSON 数据结构。

用户价值：

- 适合深入分析调用树和时间线。
- 页面交互丰富，能切换视图和选项。

代码引用：

```python
if len(session.frame_records) > 100_000:
    ...
    session = original_session.resample(interval=resample_interval)
```

```python
page = f"""<!DOCTYPE html>
    <html>
    ...
    <script>
        const sessionData = {session_json};
        pyinstrumentHTMLRenderer.render(document.getElementById('app'), sessionData);
    </script>
"""
```

### JSON 渲染

对应文件：`pyinstrument/renderers/jsonrenderer.py`

功能点：

- 将会话渲染为 JSON 树结构。
- 包含函数名、文件路径、行号、时间、await 时间、子节点、分组信息、类名等字段。
- 避免直接依赖标准 `json` 递归编码深栈，降低深层调用栈崩溃风险。

适用场景：

- 自动化工具链。
- 外部系统对结果做二次处理。
- 测试与比对输出结构。

### Speedscope 渲染

对应文件：`pyinstrument/renderers/speedscope.py`

功能点：

- 生成符合 [Speedscope](https://www.speedscope.app/) 规范的事件流 JSON。
- 构建 open/close 事件序列。
- 维护 frame 去重映射表，保证共享帧索引稳定。
- 支持事件型性能火焰图查看。

适用场景：

- 使用外部 Speedscope 工具进行可视化分析。
- 与其他 profile 工具生态兼容。

### pstats 渲染

对应文件：`pyinstrument/renderers/pstatsrenderer.py`

功能点：

- 将性能数据转为 `pstats` 兼容格式。
- 输出 marshal 编码后的 stats 字典。
- 适配 gprof2dot、snakeviz 等常见工具。
- 跳过 synthetic 节点，只保留有意义的真实调用统计。

### Session 渲染

对应文件：`pyinstrument/renderers/session.py`

功能点：

- 将 `Session` 序列化为 `.pyisession` 风格 JSON。
- 用于保存、重载、回放已有 profile 结果。

### 渲染器统一导出

对应文件：`pyinstrument/renderers/__init__.py`

功能点：

- 对外统一暴露 `ConsoleRenderer`、`HTMLRenderer`、`JSONRenderer`、`PstatsRenderer`、`SessionRenderer`、`SpeedscopeRenderer` 等公开类。
- 简化调用端导入方式。

### 调用树处理器

对应文件：`pyinstrument/processors.py`

功能点：

- `remove_importlib`：去除 importlib 噪声调用。
- `remove_tracebackhide`：删除带 `__tracebackhide__` 的帧。
- `aggregate_repeated_calls`：聚合重复调用。
- `group_library_frames_processor`：将库代码分组隐藏。
- `merge_consecutive_self_time`：合并连续 self-time 节点。
- `remove_unnecessary_self_time_nodes`：删除无意义 self-time 节点。
- `remove_irrelevant_nodes`：过滤耗时占比很低的节点。
- `remove_first_pyinstrument_frames_processor`：去除 CLI 自身启动栈。

这些处理器是渲染模块的“中间层逻辑”，直接决定输出结果的可读性。

## 输入输出需求

### 输入

- `Session` 对象
- `Frame` 调用树
- 渲染参数，例如：
  - `show_all`
  - `timeline`
  - `flat`
  - `time`
  - `resample_interval`
  - `processor_options`

### 输出

- 纯文本字符串
- HTML 字符串或 HTML 文件
- JSON 字符串
- Speedscope JSON
- pstats 字符串
- `.pyisession` JSON

## 关键非功能需求

### 正确性

- 输出必须能准确反映 `Session` 中的时间与层级关系。
- 过滤、隐藏、聚合等处理不能破坏整体时间一致性。

### 可读性

- 文本输出应清晰展示层级结构、时间、代码位置。
- HTML 输出应支持交互式分析。

### 稳定性

- 空 profile 不应崩溃。
- 大 profile 应可自动重采样。
- 缺失 HTML 资源时应抛出可理解异常。
- 不同平台路径处理应尽量稳定。

### 兼容性

- 兼容终端、浏览器、JSON 消费端、Speedscope、pstats 工具链。
- 兼容 Windows、Linux、macOS 的路径差异。

## 风险点

- 路径缩短逻辑在不同平台上的结果不同，容易影响隐藏库名的显示。
- 深层递归调用树可能导致序列化或渲染性能问题。
- HTML 前端资源缺失会直接导致页面不可用。
- 过大的 `frame_records` 会导致浏览器性能问题，需要依赖重采样逻辑。

## 本模块在系统中的位置

性能分析主流程可以概括为：

1. `Profiler` 采样生成 `Session`。
2. `Session.root_frame()` 构建调用树。
3. `processors` 对调用树做预处理。
4. `renderers` 输出为用户需要的格式。

因此，输出与渲染模块是性能分析结果面向最终用户的直接出口，也是测试中最需要关注“用户可见行为”的部分。
